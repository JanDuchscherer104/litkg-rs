# ISSUE-0009 Native Viewer MVP

## Title

Add a native Rust viewer and exploration tool optimized for Apple Silicon

## Summary

Implement the first repo-owned native graph inspection path for `litgraph-rs` using a deterministic local viewer over the Neo4j export bundle and wire it into the CLI as an operator-facing inspection command.

## Motivation

- The current practical inspection surface is still the consumer repo's `graphify-out/graph.html`, which keeps the primary exploration path browser-bound and outside this repo's ownership.
- `litgraph-rs` already emits a stable export bundle under `neo4j-export/`, which is a cleaner repo-owned contract for a native viewer than relying on Graphify internals.
- The architecture direction already prefers a native Rust viewer, with `egui + eframe + petgraph` as the practical default for a first implementation.

## Current Scope

1. Add a repo-owned viewer crate built on `eframe`/`egui`/`petgraph`.
2. Load and inspect the existing `nodes.jsonl` / `edges.jsonl` export bundle.
3. Provide deterministic layout, search, selection, and metadata inspection.
4. Add a `litkg inspect-graph --config ...` command that exports on demand and launches the viewer.
5. Keep the implementation repo-independent and avoid Graphify- or consumer-specific assumptions.

## Acceptance Criteria

- A local operator can launch a native graph inspector from `litkg-cli`.
- The viewer opens a Neo4j export bundle generated by `litgraph-rs` without requiring Neo4j itself.
- The first iteration supports pan/zoom, search, node selection, and side-panel metadata inspection.
- The implementation remains deterministic and repo-independent.
- The viewer path compiles and is covered by non-GUI tests for bundle loading and core layout behavior.
